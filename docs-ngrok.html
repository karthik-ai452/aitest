Sharing API Keys with ngrok

A complete guide to securely exposing your local API to the internet using ngrok, so you can share API keys and test webhooks with external services. ← Back to Documentation

In this guide

What is ngrok?

ngrok is a tool that creates secure tunnels from a public URL to your local machine. It lets you expose a local server running on your computer to the internet — no firewall changes, no port forwarding, no static IP needed.

This is especially useful when you need to share a local API with teammates, test webhooks from third-party services (Stripe, GitHub, Slack), or let someone access your development environment remotely.

1

Install ngrok

Choose the installation method for your operating system:

🍎 macOS (Homebrew)

brew install ngrok/ngrok/ngrok

🐧 Linux (apt)

curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc \ | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc > /dev/null echo "deb https://ngrok-agent.s3.amazonaws.com buster main" \ | sudo tee /etc/apt/sources.list.d/ngrok.list sudo apt update sudo apt install ngrok

🪟 Windows (Chocolatey)

choco install ngrok

📦 Manual (any OS)

# 1. Download from https://ngrok.com/download # 2. Unzip the binary unzip /path/to/ngrok.zip # 3. Move to a PATH location (optional) sudo mv ngrok /usr/local/bin/
check_circle

Verify installation

Run ngrok version in your terminal. You should see the version number printed.

2

Sign up & authenticate

ngrok requires a free account to use. Sign up at dashboard.ngrok.com/signup and then connect your authtoken:

# Get your authtoken from: https://dashboard.ngrok.com/get-started/your-authtoken ngrok config add-authtoken YOUR_AUTH_TOKEN_HERE

This stores your token in the ngrok config file at ~/.ngrok2/ngrok.yml (or the newer ~/.config/ngrok/ngrok.yml). You only need to do this once.

3

Start a tunnel

First, make sure your local API server is running. For example, if your app is on port 3000:

# Start ngrok tunnel to your local port 3000 ngrok http 3000

You'll see a terminal UI like this:

ngrok Session Status online Account your-email@example.com Version 3.x.x Region United States (us) Latency - Web Interface http://127.0.0.1:4040 Forwarding https://a1b2c3d4.ngrok-free.app -> http://localhost:3000 Connections ttl opn rt1 rt5 p50 p90 0 0 0.00 0.00 0.00 0.00
check_circle

The forwarding URL is your public address

Copy the https://a1b2c3d4.ngrok-free.app URL — this is what you share with others to reach your local API.

You can also use the Web Interface at http://127.0.0.1:4040 to inspect all incoming requests and responses in real time.

4

Share your API with others

Now that your tunnel is running, anyone with the ngrok URL can hit your local API. Share it like this:

# Your public API base URL (share this) https://a1b2c3d4.ngrok-free.app # Example: If your local API has an endpoint /api/health curl https://a1b2c3d4.ngrok-free.app/api/health # Example: POST with an API key header curl -X POST https://a1b2c3d4.ngrok-free.app/api/test \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{"test": "data"}'

Share the URL with your teammates, clients, or third-party services. They can use it just like any public API.

5

Secure your tunnel

Since your tunnel is publicly accessible, you should add protection:

Basic Authentication

ngrok http 3000 --basic-auth="username:password"

Visitors will be prompted for a username and password before they can access your API.

IP Whitelisting

ngrok http 3000 --allow-ip-cidr="203.0.113.0/24"

Only allow connections from specific IP addresses or CIDR ranges.

OAuth (Google / GitHub / Microsoft)

ngrok http 3000 --oauth=google

Requires visitors to sign in with a Google (or GitHub, Microsoft) account before accessing the tunnel. Available on paid plans.

warning

Never leave tunnels open unattended

Always stop your ngrok tunnel (Ctrl+C) when you're done. An open tunnel exposes your local machine to the internet.

6

Share & test API keys

Once your tunnel is running, you can share API keys so others can authenticate against your local service:

# 1. Generate an API key in your local app curl -X POST http://localhost:3000/api/keys \ -H "Content-Type: application/json" \ -d '{"name": "test-key-for-teammate"}' # Response: # { "key": "sk_test_abc123..." } # 2. Share the ngrok URL + the API key with your teammate # They can now make authenticated requests: curl https://a1b2c3d4.ngrok-free.app/api/run-test \ -H "Authorization: Bearer sk_test_abc123..." \ -H "Content-Type: application/json" \ -d '{"suite": "E2E-SMOKE", "env": "staging"}'
shield

Best practice

Use environment variables to store API keys — never hardcode them. Use .env files locally and inject them via your hosting platform in production.

Troubleshooting

error "Tunnel session failed: Authorization failed"

Make sure you've added your authtoken: ngrok config add-authtoken YOUR_TOKEN

error "ERR_NGROK_108: The ngrok account is limited to 1 simultaneous session"

Free accounts are limited to one tunnel at a time. Stop any existing tunnel (Ctrl+C) before starting a new one, or upgrade to a paid plan.

warning URL changes every time I restart ngrok

Free plans generate a random URL each session. For a stable subdomain, use a paid plan: ngrok http 3000 --domain=myapp.ngrok-free.app

warning "Connection refused" when accessing the URL

Your local server isn't running or is on a different port. Verify it's running: curl http://localhost:3000

For more details, visit the official ngrok documentation.
