Sharing API Keys with ngrok

A complete guide to securely exposing your local API to the internet using ngrok.

1. What is ngrok?

ngrok is a tool that creates secure tunnels from a public URL to your local machine. It's perfect for testing webhooks, sharing local APIs, and demoing projects without deploying.

2. Installation

Install ngrok via your package manager or download directly:

# macOS
brew install ngrok

# Linux (snap)
snap install ngrok

# Or download from https://ngrok.com/download

Then authenticate with your ngrok account:

ngrok config add-authtoken YOUR_AUTH_TOKEN

3. Basic Usage

Expose a local port to the internet:

# Expose local server on port 8000
ngrok http 8000

ngrok will display a forwarding URL like:

Forwarding  https://abc123.ngrok-free.app -> http://localhost:8000

4. Sharing API Keys

Use ngrok to expose your local CmdRunner API so external services (like Paddle, Slack, or GitHub) can reach your webhook endpoints. 

# Start your local API
python -m cmdrunner_web --port 8000

# In another terminal, create a tunnel
ngrok http 8000

Use the ngrok forwarding URL as your webhook endpoint:

# Example webhook URL
https://abc123.ngrok-free.app/webhook/paddle

5. Security Best Practices

  • Never share your ngrok authtoken publicly
  • Use webhook signature verification on all incoming requests
  • Close ngrok tunnels when not in use
  • Use ngrok's built-in basic auth for sensitive endpoints: ngrok http 8000 --auth="user:pass"

6. Troubleshooting

"Tunnel session failed"

Make sure your local server is running before starting ngrok. Check the port matches.

"Account limited"

Free ngrok accounts have limits. Upgrade your plan or use a custom domain.

Connection refused

Your local service isn't listening on the port ngrok is forwarding to. Verify the port number.
